# Contributing

Thanks for your interest in contributing.

## Getting started

```bash
bun install
bun run typecheck
bun run build
```

## Code style

- Use English for docs and commit messages.
- Keep changes small and focused.
- Do not include secrets in code or logs.
- Avoid unrelated refactors.

## Pull request checklist

- [ ] Scope and behavior are described clearly
- [ ] Docs updated for user-visible changes
- [ ] No secrets added to the repository
- [ ] Validation output included in PR description

## Reporting issues

Use issue templates and include reproduction steps, environment details, and expected/actual behavior.
